Guia Interativo — GitHub

Manual prático para criar e gerir um repositório no GitHub, organizar trabalho, colaborar (branches + pull requests) e publicar um site no GitHub Pages.

Exemplo aplicado: organização de uma PAP (documentação + versões + entrega).

🌐 GitHub ✅ Passo a passo 🧩 Exemplos + prints 🚀 Publicar no Pages

Antes de começar

Conta, email verificado e um repositório para a PAP.

Checklist
Tenho conta no GitHub
Se ainda não tiveres: criar conta e confirmar email.
Perfil atualizado
Nome + foto (opcional) ajudam na identificação do trabalho.

Regras rápidas (boa prática)

Deixa o repositório “limpo” e fácil de avaliar.

Boas práticas
  • README com objetivo, estrutura e links.
  • Pastas bem organizadas: docs/, src/, images/.
  • Commits com mensagens claras (ex.: “Adicionar introdução ao relatório”).
  • Usar Issues para tarefas e Pull Requests para integrar mudanças.

1) Conta & Perfil

Configurar a conta e deixar o perfil pronto para usar o GitHub em projeto.

1

Criar conta / Confirmar email

Sem email confirmado podes ter limitações em algumas ações.

Link: Criar conta GitHub
Depois: Settings → Emails → confirmar/verificar.
2

Atualizar perfil

Adicionar nome e (opcional) foto para identificar o trabalho.

Link: Settings → Profile

2) Criar repositório

Criar o repositório onde vais guardar a PAP (código + docs + recursos).

1

Nome do repositório

Escolhe um nome simples e claro.

Sugestões: pap, pap-projeto, projeto-pap, pap-2025
Link: Create a new repository
2

Configurações recomendadas

Public (para Pages) e README ligado.

✅ Visibility: Public
✅ Add README: ON
(Opcional) Add .gitignore / License
📸 Print sugerido: ecrã “Create a new repository”
Coloca o teu print em images/ e atualiza o src.
Ex.: images/print_create_repo.png

3) Upload e organização

Organizar a estrutura do projeto e adicionar ficheiros no GitHub.

1

Criar pastas do projeto

Estrutura recomendada (PAP).

Estrutura recomendada:
Estrutura 
pap/
  README.md
  index.html
  src/
  docs/
  images/
No GitHub: Add file → Create new file para criar pastas (ex.: docs/README.md).
2

Upload de ficheiros

Adicionar documentos, imagens e ficheiros do projeto.

No repositório: Add file → Upload files.
Boa prática: colocar prints em images/ e relatórios em docs/.

4) README e documentação

O README é a “capa” do projeto: explica o que é e como usar/avaliar.

1

Modelo de README (PAP)

Copia e adapta ao teu projeto.

README.md (modelo) 
# PAP — [Título do Projeto]

## Objetivo
Breve descrição do problema e do que o projeto pretende resolver.

## Estrutura do repositório
- `src/` — código do projeto
- `docs/` — documentação (relatório, evidências)
- `images/` — prints e imagens

## Como aceder
- Repositório: https://github.com/TEU_USER/TEU_REPO
- Website (GitHub Pages): https://TEU_USER.github.io/TEU_REPO/

## Evidências
Coloca aqui prints importantes (ou links para `images/`).
Dica: no GitHub podes editar o README clicando no ícone do lápis ✏️.

5) Branches & Pull Requests

Trabalhar em mudanças sem mexer diretamente na main e integrar com Pull Request.

1

Criar branch no GitHub

No dropdown da branch (main) cria uma nova branch.

Exemplo de branch: docs-relatorio ou pagina-contactos
No topo do repositório: selecionar branch → escrever nome → “Create branch”.
2

Abrir Pull Request

Comparar branch com main e pedir merge.

Tab Pull requests → New pull request → base: main ← compare: tua branch.
Escrever um título e descrição clara (o que mudou e porquê).
3

Merge do Pull Request

Integrar a mudança na main.

No PR: Merge pull request → Confirm merge.

6) Issues & Tarefas

Usa Issues como lista de tarefas e registo de trabalho.

1

Criar Issue para uma tarefa

Ex.: “Adicionar introdução ao relatório”

Tab Issues → New issue.
Usa checklist:
Checklist (Markdown) 
- [ ] Escrever introdução
- [ ] Inserir objetivos
- [ ] Adicionar bibliografia

7) GitHub Pages

Publicar um site a partir do teu repositório (ex.: manual, relatório, demonstração).

1

Ativar Pages

Settings → Pages → Deploy from branch → main / root.

Depois aparece um link do tipo: https://TEU_USER.github.io/TEU_REPO/
O GitHub abre automaticamente o index.html.

8) Downloads para o projeto

Links úteis para instalar ferramentas e descarregar o repositório.

1

Links oficiais

Aqui estão os links mais usados.

✅ GitHub (conta): https://github.com/join
✅ GitHub Desktop: https://desktop.github.com/
✅ Git (download): https://git-scm.com/downloads

Como descarregar o repositório

Para ter uma cópia do projeto no computador.

No repositório: botão Code → Download ZIP.
Alternativa: usar GitHub Desktop para “Clone repository”.

Checklist PAP

Lista final para garantir que o repositório está pronto para entrega/avaliação.

1

README completo

Objetivo, estrutura, links e evidências.

Inclui: objetivo, estrutura, links (repo + pages) e prints importantes.
2

Evidências (prints) organizadas

Guardar prints em images/ e referir no README.

Exemplos: criação de repo, upload, PR aberto, Pages ativado.
3

Pages publicado (se aplicável)

Site a abrir corretamente e link no README.

Confirma o link do Pages e que o index.html abre bem.
Entrega sugerida: link do repositório + link do Pages + PDF com prints (opcional).
