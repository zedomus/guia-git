Guia Interativo — Git/GitHub para a PAP

Vais aprender o essencial: commits, recuperação e branches. No final, tens um mini-desafio com boas práticas para aplicares na tua PAP.

🧑‍🎓 11.º Ano 🐧 Linux 🌐 GitHub ✅ Passo a passo + prints

Antes de começar

Confirma que tens Git instalado e conta no GitHub.

Checklist
Tenho Git instalado
Comando: git --version
Tenho conta GitHub
Vou conseguir criar repositórios e ver commits online.

Regras de ouro (PAP)

Pequenas rotinas que evitam dores de cabeça.

Boas práticas
  • Commit frequente com mensagens claras.
  • Antes de mudanças grandes: cria branch.
  • Se algo correr mal: não apagues a pasta — recupera com Git.
  • Trabalha com ficheiros de texto (código) e ignora lixo com .gitignore.

1) O básico do Git

Criar repositório, adicionar ficheiros e fazer commits (histórico de versões).

1

Criar pasta do projeto (PAP)

Cria uma pasta e entra nela.

Terminal Linux 
mkdir projeto_pap
cd projeto_pap
Se a pasta já existir, usa só cd projeto_pap.
2

Inicializar repositório Git

Cria a pasta oculta .git (histórico).

Terminal Linux 
git init
📸 Print obrigatório (exemplo): git init executado
Como inserir prints: cria a pasta images/ no repositório e faz upload dos PNG/JPG.
Exemplo de nome: images/github.png
3

Primeiro ficheiro + primeiro commit

Criar README.md, adicionar e registar no histórico.

Terminal Linux 
echo "# PAP — Projeto" > README.md
git add README.md
git commit -m "Adicionar README inicial"
Dica de avaliação: mensagens de commit devem explicar a mudança (verbo + objeto).
4

Ver o histórico

Confirmar que os commits ficaram registados.

Terminal Linux 
git log --oneline --decorate

2) Processo de recuperação de um ficheiro

Como voltar atrás quando apagaste ou estragaste um ficheiro.

1

Simular um erro (estragar o ficheiro)

Vamos alterar o README e depois recuperar.

Terminal Linux 
echo "Linha errada..." >> README.md
git status
Deverás ver o ficheiro como modified no git status.
2

Recuperar o ficheiro para a versão do último commit

Descartar alterações locais (sem commit).

Opção A (recomendada) 
git restore README.md
Se o teu Git for mais antigo, podes usar: git checkout -- README.md
3

Recuperar um ficheiro apagado

Apagar e recuperar a partir do último commit.

Terminal Linux 
rm README.md
git status
git restore README.md

3) Branches & Merge

Trabalhar em funcionalidades sem estragar a versão principal.

1

Criar branch para uma funcionalidade

Ex.: criar uma página de contactos.

Terminal Linux 
git switch -c pagina_contactos
Alternativa: git checkout -b pagina_contactos
2

Adicionar ficheiro na branch

Criar contactos.html e fazer commit.

Terminal Linux 
printf "<h1>Contactos</h1>\n<p>Email: ...</p>\n" > contactos.html
git add contactos.html
git commit -m "Adicionar página de contactos"
3

Voltar à main e fazer merge

Juntar a funcionalidade à versão principal.

Terminal Linux 
git switch main
git merge pagina_contactos
Se houver conflito: lê as marcas <<<<<<< e escolhe a versão correta.

4) GitHub & GitHub Pages

Colocar o repositório online e publicar o site do projeto.

1

Criar repositório no GitHub

No GitHub: New repository → nome: pap (ou semelhante).

Depois copia o URL do repositório (HTTPS), ex.: https://github.com/teuuser/pap.git
2

Ligar repositório local ao GitHub e enviar

Primeiro push (só uma vez).

Terminal Linux 
git remote add origin URL_DO_TEU_REPO
git branch -M main
git push -u origin main
3

Publicar no GitHub Pages

Settings → Pages → Deploy from branch → main / root

O teu site ficará em: https://teuuser.github.io/teurepo/

Desafio PAP

Tarefa final para entregar (podes anexar prints e link do GitHub Pages).

1

Organização do repositório

Estrutura mínima recomendada para PAP.

Estrutura sugerida 
pap/
  README.md
  index.html
  src/         (código)
  docs/        (documentação)
  images/      (prints e imagens)
  .gitignore
2

3 commits “bons”

Faz 3 commits com mensagens claras.

Exemplos de mensagens:
"Criar estrutura do projeto" • "Adicionar página inicial" • "Corrigir layout e links"
3

Link final

Entrega o link do repositório e do GitHub Pages.
