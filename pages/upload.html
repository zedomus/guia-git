Upload & Organização

Nesta página vais criar uma estrutura organizada e fazer upload de ficheiros. A ideia é o repositório ficar “limpo” e fácil de navegar (exemplo: PAP).

📁 Pastas ⬆️ Upload 🧾 Evidências

Estrutura recomendada

Sugestão simples e eficiente para projetos com documentação + evidências.

Estrutura 
src/       (código)
docs/      (documentação / relatório)
images/    (prints e imagens)
No GitHub, criar pastas via interface: Add file → Create new file e escreve, por exemplo: docs/README.md (o GitHub cria a pasta automaticamente).

Passos

Segue os passos e coloca prints (opcional) como evidência.

1

Criar as pastas no repositório

Cria src/, docs/ e images/.

Forma rápida: cria um ficheiro dentro de cada pasta:
  • src/.keep
  • docs/.keep
  • images/.keep
(Depois podes apagar os .keep quando já tiveres ficheiros reais.)
2

Fazer upload de ficheiros

Add file → Upload files (para docs, imagens e recursos).

Sugestão de destino:
  • Relatório/ficheiros PDF: docs/
  • Imagens/prints: images/
  • Ficheiros de projeto: src/
📸 Print sugerido: ecrã “Upload files” antes do commit
📸 Print sugerido: lista de ficheiros no repo depois do upload
3

Mensagem de commit clara

Descreve o que adicionaste.

Exemplos 
Adicionar estrutura inicial do projeto
Adicionar documentação em docs/
Adicionar prints de evidência em images/
Dica: Quando tiveres o Pages ativo, podes colocar no docs/ ou no site uma secção “Evidências” com links para as imagens.
