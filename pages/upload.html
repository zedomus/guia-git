Upload & Organização

Organizar o repositório (estrutura de pastas) e carregar ficheiros e evidências (prints, docs, etc.).

📁 Estrutura ⬆️ Upload 🧾 Evidências

Estrutura recomendada (exemplo PAP)

Mantém tudo organizado para facilitar trabalho e avaliação.

1

Criar pastas essenciais

Separar código, documentação e imagens.

Estrutura:
Estrutura 
src/      (código)
docs/     (relatório / documentação)
images/   (prints e imagens)
No GitHub, para criar pastas via interface: Add file → Create new file e escreve, por exemplo: docs/README.md (o GitHub cria a pasta automaticamente).
2

Fazer upload de ficheiros

Adicionar documentos, imagens e recursos.

No repositório: Add file → Upload files.
Sugestão:
  • Relatório → docs/
  • Imagens/prints → images/
  • Código → src/
3

Commit message clara

Ao fazer upload, escreve uma mensagem que explique o que mudou.

Exemplos:
Mensagens 
Adicionar estrutura inicial do projeto
Adicionar prints de evidência
Atualizar documentação no docs/
Prints sugeridos: (1) árvore de pastas no GitHub, (2) ecrã de upload concluído.
