Issues (tarefas, evidências e organização)

As Issues são uma das melhores formas de manter uma PAP organizada. Funcionam como lista de tarefas, registo de decisões e prova de progresso ao longo do tempo.

A ideia é simples: tudo o que “falta fazer” vira uma Issue. Quando resolves, atualizas a Issue (checklist, comentários, anexos/links) e no fim fechas. Resultado: tens um histórico claro do que foi feito, quando e porquê.

Boa prática: usa Issues para tarefas grandes e também para “micro-tarefas” semanais. A regularidade é o que cria a melhor evidência.

Passos essenciais

Cria Issues, adiciona checklist, liga ao trabalho e fecha com evidência.

Infográfico: passos essenciais (criar Issue → checklist → labels → fechar)
1

Criar uma Issue

Repositório → separador Issues → New issue.

Exemplos de títulos “bons”:
  • Relatório: Introdução e objetivos
  • Implementação: Criar menu e páginas base
  • Testes: Validar formulário / corrigir erros
  • Documentação: Atualizar evidências semanais
2

Adicionar checklist (Markdown)

Transforma a Issue num mini-plano com tarefas concretas.

Checklist (copiar e adaptar) 
- [ ] Definir objetivo desta parte
- [ ] Fazer pesquisa / recolher fontes
- [ ] Implementar / escrever
- [ ] Rever e corrigir
- [ ] Adicionar evidências (prints/links)
Dica: no final, coloca na Issue links diretos para ficheiros do repo (ex.: PDF em docs/) e para prints em images/.
3

Usar Labels para organizar

Labels ajudam a separar fases e prioridades (ex.: documentação, código, urgente).

Sugestão de labels para PAP:
  • documentação • código • testes • design
  • urgente • importante • opcional
  • semana-1, semana-2…
4

Ligar Issues ao trabalho (commits e PR)

Usa Closes #N no PR para fechar automaticamente a Issue.

Texto exemplo (PR) 
Closes #12

Resumo:
- Adicionada página de contactos
- Atualizado menu
- Incluídas evidências em images/
Infográfico: fechar Issues automaticamente (PR/commit)
5

Fechar a Issue (com evidência)

Antes de fechar, garante checklist completa e links para evidências.

Sumário visual (bom para evidência final)
Prints recomendados (mínimo): 1) criação da Issue, 2) checklist, 3) labels, 4) fecho (manual ou automático), 5) lista de Issues abertas/fechadas.

Modelo de Issue (pronto a copiar)

Cria uma Issue bem organizada em 30 segundos.

Infográfico: modelo de Issue (pronto a copiar)
Template 
## Objetivo
(Explica em 1-2 linhas o que queres concluir.)

## Critérios de sucesso
- [ ] O resultado final está funcional / completo
- [ ] Foi testado (breve nota)
- [ ] Evidências adicionadas (links/prints)

## Tarefas
- [ ] Tarefa 1
- [ ] Tarefa 2
- [ ] Tarefa 3

## Evidências / Links
- Documento: docs/...
- Print: images/...
- (Opcional) PR/commit: link
Se quiseres ainda mais “profissional”: usa sempre o mesmo padrão de títulos. Ex.: Relatório: ... • Implementação: ... • Testes: ...
