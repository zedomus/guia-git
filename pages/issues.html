Issues (tarefas e registo de trabalho)

Usa Issues como lista de tarefas, evidência de progresso e organização do projeto (ex.: PAP).

✅ Tarefas 🗂️ Organização 📌 Evidência

Passos

Cria Issues com checklist e fecha quando terminar.

1

Criar uma Issue

Tab Issues → New issue.

Título exemplo: Relatório — Introdução e objetivos
2

Adicionar checklist na descrição

Ajuda a acompanhar o que falta fazer.

Checklist (Markdown) 
- [ ] Escrever introdução
- [ ] Definir objetivos
- [ ] Adicionar bibliografia
- [ ] Inserir evidências (prints)
3

Fechar a Issue

Quando terminares, marca como concluída (Close issue).

Dica: Issues fechadas são ótima evidência de trabalho organizado.
Print sugerido: lista de Issues (abertas e fechadas).
