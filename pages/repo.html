Criar repositório

Nesta etapa vais criar o repositório principal do projeto (ex.: PAP). Um repositório bem configurado desde o início evita confusões com versões, ajuda a organizar ficheiros e facilita a publicação no GitHub Pages (se for necessário).

No final desta página deves ter: um repositório criado, com nome e descrição, um README mínimo, e (opcional) um .gitignore para impedir “lixo” (ficheiros temporários) de aparecer no projeto.

📦 Repositório 📝 README 🧹 .gitignore 🌐 Public / Private

Antes de criar o repositório

Duas decisões simples que evitam problemas mais tarde.

Nome do repositório

Curto, claro e fácil de partilhar.

Naming
  • Boas opções: pap, projeto-pap, pap-2025
  • Evita espaços (usa hífen -).
  • Evita nomes genéricos tipo trabalho ou teste.

Visibilidade

Public ou Private — depende do objetivo.

Acesso
  • Public: necessário/recomendado se vais usar GitHub Pages publicamente.
  • Private: melhor para trabalho fechado (mas pode limitar Pages, dependendo do plano).
  • Mesmo em Public, não publiques dados pessoais em documentos/prints.
Regra simples: se vais precisar de um link público para mostrar resultados (site/portefólio), escolhe Public. Se o repositório for só para organização interna e não queres expor conteúdos, escolhe Private.

Passos (criação do repositório)

Segue por ordem e marca como concluído.

1

Abrir “New repository”

Criar um repositório novo no GitHub.

Link direto: github.com/new
Dica: confirma que estás na conta correta (canto superior direito).
📸 Print sugerido: página “Create a new repository” (início)
✅ Dica: em prints, evita mostrar emails completos/dados sensíveis (se necessário, tapa a zona).
2

Nome + descrição

Identifica o projeto e o objetivo do repositório.

Nome: curto (ex.: projeto-pap)
Descrição: uma linha clara (ex.: “PAP — aplicação web + documentação”)
📸 Print sugerido: nome e descrição preenchidos
✅ Boa prática: usa a descrição para dizer o tema/área do projeto numa frase.
3

Configurações recomendadas

Preparar o repositório para documentação e (opcionalmente) Pages.

✅ Visibility: Public (recomendado se vais publicar no Pages)
✅ Add a README file: ON
✅ (Opcional) Add .gitignore: escolhe a tecnologia (ex.: Node, Python, VisualStudioCode)
✅ (Opcional) License: pode ficar “None” para trabalhos escolares
📸 Print sugerido: opções selecionadas (README ON, visibilidade, etc.)
✅ Dica: se não souberes o .gitignore, podes adicionar mais tarde (sem problema).
4

Criar o repositório

Finalizar e entrar no repositório recém-criado.

Clica em Create repository e confirma que entraste na página do repositório.
📸 Print sugerido: página principal do repositório (ficheiros + About)
✅ Confirma: aparece o nome do repositório, o separador “Code” e o README (se ativaste).
5

Editar o README (mínimo recomendado)

O README é a “capa” do teu projeto e ajuda na avaliação/apresentação.

No repositório: abre README.md → clica no ícone do lápis (Edit).
Sugestão de conteúdo mínimo (podes copiar e adaptar):
Template README (exemplo PAP) 
# Projeto PAP — [título do projeto]

## Objetivo
Explica em 2–4 linhas o que o projeto faz e para que serve.

## Tecnologias
- (ex.: HTML/CSS/JS, Python, etc.)

## Estrutura do repositório
- `src/` — código
- `docs/` — documentação (planeamento/relatório)
- `images/` — evidências (prints)

## Estado do projeto
- Em desenvolvimento

## Links
- Repositório: (este link)
- GitHub Pages (se aplicável): (link)
📸 Print sugerido: README editado (secções visíveis)
✅ Dica: um README bem feito melhora muito a perceção do trabalho (organização e clareza).
Entregável sugerido (evidências): 1) print do ecrã “Create a new repository” com as opções, 2) print do repositório criado, 3) print do README preenchido.
